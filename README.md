## Why PomodoroJam?

Solo focus is hard. Shared accountability makes it stick.

PomodoroJam is like Spotify Jam, but for deep work. One person starts a 25-minute focus timer and shares a link. Everyone who joins sees the same timer, in perfect sync. When it ends, everyone breaks together. No meetings. No Slack. Just shared focus.

## Features

- 🎯 **Synchronized timer** — clock-based, drift-resistant. Everyone is on the same second.
- 🎸 **Jam Mode** — toggle between "host controls" and "everyone controls" like Spotify's Jam vs Listen Along
- 🔊 **Ambient sounds** — brown noise, white noise, pink noise, rain. Generated by Web Audio API — no files, no CDN.
- 👥 **Live participants** — see who's focusing with you via real-time presence
- 🔔 **Break notifications** — browser push notification when your session ends
- 📊 **Profiles & streaks** — track total pomodoros, focus hours, and daily streaks
- 🔗 **Share anywhere** — WhatsApp, Telegram, Discord, Slack, Twitter/X, and more
- 👋 **No account needed** — start and host a session as a guest

## Getting Started

**1. Clone and install**
```bash
git clone https://github.com/your-username/pomodorojam.git
cd pomodorojam
npm install
```

**2. Set up environment variables**
```bash
cp .env.local.example .env.local
```

Add your Supabase credentials to `.env.local`:
```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

**3. Set up the database**

Run `supabase/migrations/001_init.sql` in your Supabase SQL editor, then:
```sql
ALTER TABLE public.sessions ADD COLUMN IF NOT EXISTS jam_mode boolean default false;
```

Enable GitHub and/or Google OAuth in Supabase → Auth → Providers.

**4. Run locally**
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

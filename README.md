## Why PomodoroJam?

Solo focus is hard. Shared accountability makes it stick.

One person starts a 25-minute focus timer and shares a link. Everyone who joins sees the same timer, in perfect sync. When it ends, everyone breaks together. No meetings. No distractions. Just shared focus.

## How to Use

### 1. Start a session

Go to the home page and click **Start a session**. You'll get two choices:

- **Host a session** — creates a new timer and gives you a unique link to share
- **Join a session** — paste a link or session ID that someone sent you

No account needed to start or join.

### 2. Share the link

Once inside a session you'll see an **Invite** button. Click it to copy the session link or use your phone's native share sheet. Send it to anyone — they click the link and instantly join your session already in sync.

### 3. Pick your mode (Host only)

As the host you control how the timer is managed. The toggle in the bottom row lets you switch between two modes:

| Mode | Who controls the timer |
|---|---|
| **Host** | Only you can start, pause, or skip. Everyone else follows along in sync. |
| **Jam** | Everyone in the session controls the timer together — like a group decision. |

Switch between them at any time, even mid-session. The change is instant for all participants.

### 4. Focus together

Hit **Play** to start the 25-minute focus timer. Everyone in the session sees the same countdown, on the same second. When the timer ends:

- A sound plays and a browser notification fires for everyone
- A break overlay appears — dismiss it to start the next round

### 5. Customize your session (Host only)

Click the **⚙️ gear icon** to open timer settings:

- Set custom durations for Focus, Short Break, and Long Break
- Set the number of rounds before a long break
- Toggle **Allow guests to invite** — if off, only you can share the session link

Settings take effect immediately and reset the current timer.

### 6. Ambient sound

Click **🎵** to toggle focus noise. Choose from:

- Brown noise
- White noise
- Pink noise
- Rain

All sounds are generated in the browser — no files downloaded, no CDN.

### 7. Sign in for stats (optional)

You can use PomodoroJam entirely as a guest. If you sign in with GitHub or Google you unlock:

- **Profile page** — public page with your username
- **Total pomodoros** completed
- **Total focus hours** logged
- **Daily streak** tracking

Your stats update automatically every time a focus session completes while you're signed in.

## Features

- 🎯 **Synchronized timer** — clock-based, drift-resistant. Everyone is on the same second.
- 🎸 **Jam Mode** — toggle between "Host controls" and "everyone controls" directly inside the session
- 🔊 **Focus Noise** — brown, white, pink noise and rain. Generated by Web Audio API — no files, no CDN.
- 👥 **Live participants** — see who's focusing with you via real-time presence
- 🔔 **Break notifications** — browser push notification when your session ends
- 📊 **Profiles & streaks** — track total pomodoros, focus hours, and daily streaks
- 🔗 **One-tap share** — copy link or use the native OS share sheet
- 👋 **No account needed** — start and join sessions as a guest

## Getting Started

**1. Clone and install**

```bash
git clone https://github.com/MinitJain/pomodoro-jam.git
cd pomodoro-jam
npm install
```

**2. Set up environment variables**

```bash
cp .env.local.example .env.local
```

Fill in your Supabase credentials:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

**3. Set up the database**

Run the migrations in your Supabase SQL editor in order:

```text
supabase/migrations/001_init.sql
supabase/migrations/002_jam_mode.sql
```

Enable GitHub and/or Google OAuth in Supabase → Authentication → Providers.

**4. Run locally**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## Deploy to Vercel

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/MinitJain/pomodoro-jam)

After deploying:

1. Add your Vercel URL to Supabase → Authentication → URL Configuration → Redirect URLs
2. Add your Vercel domain to Google Cloud Console → OAuth client → Authorized JavaScript origins
3. Update `NEXT_PUBLIC_APP_URL` in Vercel environment variables
